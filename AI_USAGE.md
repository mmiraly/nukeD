# AI Usage

AI tools are allowed in nukeD development. They are tools, not authors or
reviewers.

## Expectations

If you submit AI-assisted work, you must:

- understand the change well enough to explain it without the tool
- review and edit generated text or code
- run the relevant checks
- remove speculation, filler, and unrelated changes
- disclose meaningful AI assistance in the issue or PR

Disclosure can be short:

```text
AI-assisted: used Claude/Codex/Cursor to draft the initial patch; reviewed and
tested manually.
```

## What Not To Submit

Do not submit:

- generated code you cannot explain
- long AI-written issue text that has not been edited down
- fake certainty about untested behavior
- unrelated rewrites produced while chasing a small bug
- AI-generated branding, images, audio, or video unless explicitly approved

## Maintainer Use

The maintainer may use AI tools during development. Final responsibility for
what is merged and released remains human.

## Practical Standard

The standard is simple: if a human cannot confidently review, test, and explain
the contribution, it is not ready.
